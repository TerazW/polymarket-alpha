EXECUTIVE SUMMARY
================================================================================

Belief Reaction System is a Human Belief Reaction Sensing System designed 
for event-driven prediction markets like Polymarket.

It doesn't predict price direction.
It doesn't generate trading signals.
It answers ONE question:

    "Has someone already changed their belief?"

By monitoring how order book liquidity REACTS under pressure (not its static 
existence), the system can detect belief wavering or collapse BEFORE price 
moves.

THE INSIGHT: Core Discovery
================================================================================

    "看存在没意义，看反应才有意义"
    "Observing existence is meaningless; observing REACTION is everything."

In Polymarket:
- Placing an order ≈ Belief declaration ("I believe this probability is right")
- Canceling/chasing ≈ Belief revision ("I changed my mind")
- Large resting order ≈ Belief anchor ("I strongly believe")
- Behavior after impact ≈ True belief strength ("Is this belief real or fake?")

Because there's no market maker depth protection, DOM changes ARE real human 
decision changes. This is impossible in traditional markets.

THE SOLUTION: What We Do
================================================================================

PRODUCT POSITIONING
-------------------
This is a "Human Belief Reaction Sensing System for Event-Driven Markets"

Not a trading tool.
Not order flow.
A NEW SPECIES.

CORE CAPABILITIES
-----------------
1. Detect belief changes: Monitor order book reaction under shock to judge 
   if belief is wavering
   
2. Classify reaction types: Six atomic types (HOLD/DELAY/PULL/VACUUM/CHASE/FAKE)

3. Track belief state: Each market maintains a 4-state machine 
   (STABLE/FRAGILE/CRACKING/BROKEN)
   
4. Provide verifiable evidence: Every judgment backed by liquidity data; 
   users can replay and verify on heatmap

SIX REACTION PRIMITIVES (Reaction Vocabulary)
---------------------------------------------
This is the system's LANGUAGE — all judgments expressed in this vocabulary:

┌─────────┬───────────┬─────────────────────────────────────────────────────┐
│  Type   │  Meaning  │                   Behavior                          │
├─────────┼───────────┼─────────────────────────────────────────────────────┤
│  HOLD   │  Defend   │ Quickly refills after shock; belief is firm         │
│  DELAY  │  Hesitate │ Hesitates after shock; partial refill               │
│  PULL   │  Retreat  │ Cancels immediately after shock; belief cracked     │
│  VACUUM │  Vacuum   │ Liquidity completely vanishes; possible info leak   │
│  CHASE  │  Chase    │ Anchor moves; belief is repricing                   │
│  FAKE   │  Anchor   │ Adds more after shock; psychological anchoring      │
└─────────┴───────────┴─────────────────────────────────────────────────────┘

HOW IT WORKS
================================================================================

FOUR-LAYER ARCHITECTURE
-----------------------
1. Data Layer: Receive Polymarket WebSocket data, maintain real-time state
2. Reaction Engine: Detect shocks, observe reaction window, classify reactions
3. Belief State Engine: Update market belief state based on reaction events
4. Evidence UI: Display state, alerts, heatmap evidence

BELIEF STATE MACHINE
--------------------
Each market has a real-time belief state, driven by reaction events:

┌───────────┬──────────────────────────────────┬─────────────────────┐
│   State   │            Meaning               │      Trigger        │
├───────────┼──────────────────────────────────┼─────────────────────┤
│ 🟢 STABLE │ Market belief is firm/consistent │ Consecutive HOLD/FAKE│
│ 🟡 FRAGILE│ Market belief shows weakness     │ HOLD + DELAY mix    │
│ 🟠 CRACKING│ Market belief actively breaking │ PULL/VACUUM appears │
│ 🔴 BROKEN │ Market belief has collapsed      │ Multi-level VACUUM  │
└───────────┴──────────────────────────────────┴─────────────────────┘

CORRECT POSITIONING OF CHARTS
-----------------------------
IMPORTANT PRINCIPLE: Charts are EVIDENCE layer, NOT signal source.

In this system:
- PRIMARY interface: State and reactions (text/status lights)
- SECONDARY interface: Heatmap (evidence replay)
- Users make decisions from state screen, validate on chart screen

Like medicine: You don't stare at MRI all day. You hear diagnosis first, 
then look at imaging to confirm.
