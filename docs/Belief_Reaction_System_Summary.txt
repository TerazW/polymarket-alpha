================================================================================
ONE-SENTENCE DEFINITION
================================================================================

"Human Belief Reaction Sensing System for Event-Driven Markets"

No price prediction. No trading signals.
Answers ONE question: Has someone already changed their belief?

================================================================================
CORE INSIGHT
================================================================================

    "看存在没意义，看反应才有意义"
    "Observing existence is meaningless; observing REACTION is everything."

- Polymarket is probability aggregation market, not value discovery
- Placing order = belief declaration; canceling/chasing = belief revision
- No market maker protection → DOM changes = real human decision changes
- Traditional tools (Profile/Footprint/K-line) assumptions don't hold here

================================================================================
REACTION VOCABULARY (6 Atomic Types)
================================================================================

┌─────────┬───────────┬─────────────────────────────────────────────────────────┐
│  Type   │  Meaning  │              Detection Rule                             │
├─────────┼───────────┼─────────────────────────────────────────────────────────┤
│  HOLD   │  Defend   │ refill_ratio ≥ 0.8 AND time_to_refill < 5s              │
│  DELAY  │  Hesitate │ 0.3 ≤ refill_ratio < 0.8                                │
│  PULL   │  Retreat  │ min_liq ≤ 0.1×before AND no refill                      │
│  VACUUM │  Vacuum   │ min_liq ≤ 0.05×before for entire window                 │
│  CHASE  │  Chase    │ price_shift ≠ 0 (anchor moved)                          │
│  FAKE   │  Anchor   │ size_peak after shock > liquidity_before                │
└─────────┴───────────┴─────────────────────────────────────────────────────────┘

================================================================================
DATA SOURCES
================================================================================

WebSocket Market Channel (Core):
- book: Full snapshot on subscribe + after trades
- price_change: Emitted on order add/cancel (⚠️ size = NEW value, not delta)
- last_trade_price: Emitted on trade, used for shock detection
- tick_size_change: When price extreme causes tick change

REST API (Auxiliary):
- GET /book: Snapshot recovery/verification
- Gamma /markets: Market metadata (conditionId, tokenIds, tick, etc.)
